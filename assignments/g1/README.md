### Данные

#### Исходные данные

*   Обучающая выборка: 12 553 аудиофайла от 6 спикеров (spk_A-F)
*   Валидационная выборка: 2 265 файлов от 10 спикеров (A-F + H-K)
*   Т.к в оригинале transcription в числовом формате, для обучения модели преобразовал в словесную форму с помощью num2words
*   Из словесных представлений строится символьный словарь (21 токен: буквы + пробел)

#### Загрузка и ресемплинг

*   Все аудиофайлы (wav/mp3, с разной исходной частотой) загружаются с помощью torchaudio.load
*   Приводятся к моно и ресемплятся до 16кГц (по условию)

#### Аугментация данных
(Аугментации применяются только к train)

*   random_gain для имитации разной громкости записи и чувствительности микрофона (от -6 до +6 дБ)
*   add_white_noise_snr добавление фонового шума, помех (SNR от 8 до 25 дБ, вероятность 0.2)
*   time_dropout для имитации плохого соединения с обнулением короткого сегмента (максимальная длина вырезаемого сегмента 80 мс, вероятность 0.2)
*   random_time_shift компенсация неточной обрезки пауз, небольших рассинхронов начала произнесения (сдвиг до ±40 мс, вероятность 0.2)
*   speed_perturb изменение темпа речи для улучшения понимания новых спикеров и ресемлинг через sox для сохранения целевой частоты (коэффициенты 0.9, 1.0, 1.1, вероятность 0.3)
*   NoiseMixer добавляет смешивание с сигналом со случайным SNR для имитации шума улицы, офиса и т.п (от 5 до 20 дБ, вероятность 0.3) 
*   SpecAugment после вычисления log‑Mel спектрограммы применяются две частотные маски (имитируя фильтрацию) и две времнные маски (имитируют кратковременные пропадания звука)
*   Cepstral Mean and Variance Normalization - вычисляется среднее и стандартное отклонение по времени для каждого вектора-признака, затем (feat - mean) / std для уменьшения вариаций громкости и частотной окраски


### Архитектура модели

Общая стуктура:
`log‑Mel (80) => Subsampling (Conv1d x2, stride=2) => Conformer (10 слоев) => Linear => CTC`

*   Conformer подходит для небольших моделей (self-attention + свертки, но меньше параметров, чем трансформер)
*   Subsampling через свертки, т.к. CTC требует, чтобы длина последовательности признаков была больше или равна длине целевой последовательности

#### Subsampling 

*   Первый слой: Conv1d(in=80, out=160, kernel=5, stride=2, padding=2) => ReLU => Dropout(0.1)
*   Второй слой: Conv1d(160, 160, kernel=5, stride=2, padding=2) => ReLU => Dropout(0.1)

Итого временное сжатие в 4 раза

#### Conformer encoder
из `torchaudio.models.Conformer`

*   Размерность модели d_model = 160
*   Количество слоев num_layers = 10
*   Голов внимания num_heads = 4
*   Размер FFN ffn_dim = 320
*   Размер глубинной свертки depthwise_conv_kernel_size = 31
*   Dropout везде 0.1
*   Conformer объединяет self‑attention для глобального контекста и глубинные свертки для локальной структуры речи
*   Количество параметров получилось ~ 4.1 млн

#### CTC‑head

*   Линейный слой `Linear(d_model, vocab_size=21)` с выходом логитов для каждого кадра (после subsampling)

### Пайплайн обработки
```
Аудио (разные sr) => ресемплинг 16k => waveform‑аугментации => log‑Mel + SpecAugment + CMVN => => модель => CTC‑loss => greedy‑декодинг => символы => текст => число (через ru_words_to_int)
```

Модель обучается на символьном уровне (русские буквы + пробел), а не на цифрах, что позволяет использовать лингвистическую структуру числительных
Плюс модель обобщает лучше, чем при word-уровне т.к одно и то же слово встречается в разных позициях
Преобразование из текста => цифры вынесено в постпроцессинг, чтобы модели не изучать правила формирования и склонения числительных


### Обучение модели

#### Функция потерь
*   Используется `nn.CTCLoss(blank=0, reduction='mean', zero_infinity=True)`
*   Логиты перед loss прогоняются через log_softmax (внутри loss это делается автоматически, но для стабильности `log_probs = logits.log_softmax(dim=-1)`)

#### Фильтрация коротких последовательностей (T < U)
*   После forward вычисляются out_lens (длина последовательности после subsampling + Conformer)
*   Если для какого‑то примера target_len > out_lens, CTC не может выровнять - такой пример удаляется из батча (без этого loss дает inf)
*   В случае удаления части батча forward пересчитывается заново для отфильтрованных данных

#### Оптимизация
*   Оптимизатор: AdamW (lr=2e-3, weight_decay=1e-2)
*   Scheduler: OneCycleLR - за 10% эпох повышение lr до максимального, затем косинусное затухание
*   clip_grad_norm: max norm = 5.0  (ограничение размеров градиентов для резкого изменения весов)

#### AMP
*   `torch.amp.autocast` и `GradScaler` для ускорения и экономии памяти на CUDA

#### Контроль переобучения
*   Регуляризация: Dropout (0.1) в subsampling, внутри Conformer, weight decay в AdamW
*   Аугментации (включая SpecAugment) выступают как регуляризатор
*   Отслеживание CER по каждому спикеру, в итоге на `ooD` спикерах ошибки выше, но было несколько попыток обучения с разным кол-вом эпох, чтобы модель не переобучилась на спикерах из inD

#### Метрика при сохранении
*   Итоговый скор - это гармоническое среднее CER_inD и CER_ooD (hm_cer), чтобы модель балансировала между знакомыми и незнакомыми голосами (а не просто минимизировала общую ошибку за счет улучшения метрики на inD)


### Инференс модели

1. аудио => признаки (без аугментаций, CMVN все равно применяется)
2. forward модели => логиты
3. greedy (argmax по словарю, удаление повторов, удаление blank)
4. строка символов => в число через `ru_words_to_int`
5. если парсинг не удался, то возвращается пустая строка (CER тогда будет 1.0)

В целом, предсказуемая грамматика (немного уникальных словоформ) и словарь из 21 символа, поэтому greedy должно хватать, ибо есть постобработка
За счет Conformer с subsampling и аугментациями получилось четко различать фонемы и последовательности, но для улучшения качества построцессинг:
    *   если модель пропустила "тысяча" в середине, но правильно распознала левую и правую части, ru_words_to_int может восстановить число
    *   лишние пробелы, перестановки сотен и десятков парсер исправляет благодаря жесткой логике разбора

Можно попробовать дополнительно вызывать KenLM для декодирования на оставшихся сложных примерах (spk_I самый большой CER по итогу)
